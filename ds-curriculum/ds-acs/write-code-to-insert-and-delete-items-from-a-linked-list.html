Linked List Operations

Learn how to implement insertion and deletion operations on linked lists, fundamental operations for managing dynamic data structures.

Linked List Fundamentals

Node Structure

class Node {
  constructor(data) {
    this.data = data;
    this.next = null;
  }
}

Insertion Operations

There are three main types of insertions in a singly linked list:

1. Insert at the Beginning

function insertAtBeginning(head, data) {
  // Create a new node
  const newNode = new Node(data);
  
  // Point new node to current head
  newNode.next = head;
  
  // Update head to new node
  return newNode;
}

2. Insert at the End

function insertAtEnd(head, data) {
  // Create a new node
  const newNode = new Node(data);
  
  // If list is empty, make new node the head
  if (!head) {
    return newNode;
  }
  
  // Traverse to last node
  let current = head;
  while (current.next) {
    current = current.next;
  }
  
  // Link last node to new node
  current.next = newNode;
  
  return head;
}

3. Insert at a Specific Position

function insertAtPosition(head, data, position) {
  // Create a new node
  const newNode = new Node(data);
  
  // If position is 0, insert at beginning
  if (position === 0) {
    newNode.next = head;
    return newNode;
  }
  
  // Traverse to position - 1
  let current = head;
  let count = 0;
  
  while (current && count < position - 1) {
    current = current.next;
    count++;
  }
  
  // If position is invalid
  if (!current) {
    console.log("Position out of bounds");
    return head;
  }
  
  // Insert node at position
  newNode.next = current.next;
  current.next = newNode;
  
  return head;
}

Deletion Operations

There are two main approaches to deletion in a singly linked list:

1. Delete by Position

function deleteAtPosition(head, position) {
  // If list is empty
  if (!head) {
    return null;
  }
  
  // If deleting the head
  if (position === 0) {
    return head.next;
  }
  
  // Find the node before the one to delete
  let current = head;
  let count = 0;
  
  while (current.next && count < position - 1) {
    current = current.next;
    count++;
  }
  
  // If position is invalid
  if (!current.next) {
    console.log("Position out of bounds");
    return head;
  }
  
  // Skip over the node to delete
  current.next = current.next.next;
  
  return head;
}

2. Delete by Value

function deleteByValue(head, value) {
  // If list is empty
  if (!head) {
    return null;
  }
  
  // If head has the value to delete
  if (head.data === value) {
    return head.next;
  }
  
  // Find the node before the one with value
  let current = head;
  
  while (current.next && current.next.data !== value) {
    current = current.next;
  }
  
  // If value not found
  if (!current.next) {
    console.log("Value not found");
    return head;
  }
  
  // Skip over the node to delete
  current.next = current.next.next;
  
  return head;
}

Edge Cases and Testing

Common Edge Cases

  • Empty List: Operations on an empty list should be handled appropriately
  • Single Node: Special consideration when list has only one node
  • Invalid Position: Handling positions that are out of bounds
  • Value Not Found: Handling when a value to delete doesn't exist

Testing Your Implementation

// Create a linked list: 10 -> 20 -> 30
let head = new Node(10);
head = insertAtEnd(head, 20);
head = insertAtEnd(head, 30);

// Insert at beginning: 5 -> 10 -> 20 -> 30
head = insertAtBeginning(head, 5);

// Insert at position 2: 5 -> 10 -> 15 -> 20 -> 30
head = insertAtPosition(head, 15, 2);

// Delete at position 1: 5 -> 15 -> 20 -> 30
head = deleteAtPosition(head, 1);

// Delete by value (20): 5 -> 15 -> 30
head = deleteByValue(head, 20);

// Print the final list
printList(head); // Output: 5 15 30

function printList(head) {
  let current = head;
  let output = '';
  
  while (current) {
    output += current.data + ' ';
    current = current.next;
  }
  
  console.log(output.trim());
}

Time Complexity Analysis

Insertion Operations

  • Insert at Beginning: O(1) - constant time
  • Insert at End: O(n) - need to traverse the entire list
  • Insert at Position: O(n) - worst case when position is at the end

Deletion Operations

  • Delete by Position: O(n) - worst case when position is at the end
  • Delete by Value: O(n) - need to search through the list

Space Complexity

All operations have O(1) space complexity as they only create a constant number of new variables regardless of input size.
