Overview

Module 4 shifts focus from technical preparation to career readiness, specifically building your GitHub profile and crafting an effective resume. A strong GitHub presence and well-crafted resume are essential tools for showcasing your skills and projects to potential employers in the data science field.

Through this module, you'll learn how to create a professional GitHub profile, contribute effectively to repositories, and present your work and experience in a way that attracts employer attention.

Building Your GitHub

GitHub is a crucial platform for displaying your coding projects and collaborating with others. In the tech industry, employers often look at candidates' GitHub profiles to assess their skills, coding style, and project experience.

Getting Started with GitHub

Most of you likely already have a GitHub account from previous modules. In this section, we'll focus on enhancing your GitHub presence to make it more attractive to potential employers:

  • Refreshing your GitHub profile and improving your public presence
  • Building a cohesive narrative through your repositories

Building Your Complete GitHub Profile

Make your GitHub profile career-ready with elements that will impress hiring managers:

  • Setting up a professional profile picture and bio
  • Structuring your profile to highlight your technical expertise
  • Using GitHub as a portfolio to showcase your best work

Creating Your Profile README

A Profile README is a special repository that appears at the top of your GitHub profile. It's an opportunity to showcase your skills, projects, and personality:

  • Setting up a profile README repository
  • Using Markdown to format your README
  • Including badges, images, and links
  • Highlighting your skills and projects

Pinning Projects

Pinning repositories allows you to highlight your best work:

  • Selecting projects that showcase your skills
  • Creating comprehensive README files for your projects
  • Ensuring your pinned projects are diverse and relevant
  • Updating project descriptions for clarity
Pinning Items to Your Profile

Adding Contributions to Your Profile

Keep your GitHub profile active and relevant by making regular contributions:

  • Understanding the GitHub contribution graph
  • Different ways to make contributions (commits, pull requests, issues)
  • Creating a consistent contribution schedule
  • Making meaningful contributions to your own and others' repositories

GitHub Profile Checklist

Use this checklist to ensure your GitHub profile is ready for professional presentation:

  • Professional profile picture
  • Complete bio with relevant skills and interests
  • Custom README with introduction and skills overview
  • At least 4-6 pinned repositories
  • Comprehensive README files for all pinned projects
  • Regular contribution activity (green squares on your contribution graph)
  • Contact information or link to professional profiles
  • Clean, organized code in repositories

Adding Skills

Learn how to effectively present your technical and soft skills:

  • Identifying relevant technical skills for data science roles
  • Organizing skills by proficiency level
  • Including programming languages, tools, and frameworks
  • Highlighting transferable skills from previous experiences

Focus on skills that are directly relevant to the data science positions you're targeting, and consider using action verbs to describe your proficiency levels.

Adding Projects

Projects are concrete evidence of your abilities:

  • Describing projects using the problem-solution-result format
  • Quantifying achievements and impact
  • Including links to GitHub repositories
  • Highlighting technologies and methodologies used

Experience Pre-BloomTech

Previous work experience, even if not directly related to data science, can demonstrate valuable skills:

  • Focusing on transferable skills from previous roles
  • Using action verbs to describe achievements
  • Quantifying impact wherever possible
  • Tailoring descriptions to highlight relevant experience
