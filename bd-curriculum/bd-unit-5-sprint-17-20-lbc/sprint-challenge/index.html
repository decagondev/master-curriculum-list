Sprint Challenge: Design Document

Challenge Objective

For this challenge, you'll need to complete a comprehensive design document for your Learn and Be Curious project. This document will outline the problem you're solving, your project's architecture, API design, DynamoDB table design, and frontend mockups.

Design Document Template

Below is the structure of the design document you'll need to complete. Use this template to guide your project's design and planning.

1. Problem Statement

Explain clearly what problem you are trying to solve with your project.

2. Top Questions to Resolve in Review

List the most important questions you have about your design, or things that you are still debating internally that you might like help working through.

3. Use Cases

Work backwards from the customer and define what your customers would like to do (and why). You may also include use cases for yourselves, or for the organization providing the product to customers.

Example format:

  • U1. As a [product] customer, I want to <result> when I <action>
  • U2. As a [product] customer, I want to view my grocery list when I log into the grocery list page

4. Project Scope

Clarify which parts of the problem you intend to solve.

4.1. In Scope

Which parts of the problem will you solve with this design?

4.2. Out of Scope

Are there any aspects you are not planning to solve? Identify potential expansions or related problems that you won't address in this project.

5. Proposed Architecture Overview

Describe broadly how you are proposing to solve the requirements. This may include class diagram(s) showing what components you are planning to build. Explain why your architecture represents a good data flow and separation of concerns.

6. API

6.1. Public Models

Define the data models your service will expose in its responses.

6.2. Endpoints

For each endpoint in your API, describe:

  • The behavior of the endpoint
  • What data it requires
  • What data it returns
  • How it will handle failure cases

Include a sequence diagram showing how a user interaction goes from user to website to service to database, and back.

7. Tables

Define the DynamoDB tables you will need for the data your service will use.

8. Pages

Include mock-ups of the web pages you expect to build. These can be wireframes using drawing software or hand-drawn pictures that represent the key customer-facing components of the pages.

Submission Requirements

For this sprint challenge, you'll need to:

  1. Complete the design document template
  2. Store it in your project repository
  3. Present your design for review

Your document should be comprehensive enough that someone else could implement your project based solely on your design.
