Sprint Challenge: Task Management Application

The sprint challenge is an opportunity to demonstrate your understanding of writing and testing functional requirements by building a task management application. You'll write use cases, create test cases, and implement the required functionality.

Project Overview

You'll be working on a task management application that allows users to:

  • Create new tasks
  • View all tasks
  • Mark tasks as complete
  • Delete tasks
  • Filter tasks by status
  • Search tasks by title or description

Requirements

Functional Requirements

  • Users must be able to create tasks with a title and description
  • Tasks must have a status (pending, in progress, completed)
  • Users must be able to update task status
  • Users must be able to delete tasks
  • Users must be able to filter tasks by status
  • Users must be able to search tasks by title or description

Non-Functional Requirements

  • Application must respond to user actions within 2 seconds
  • Application must handle at least 1000 tasks
  • Application must be accessible via web browser
  • Application must work on modern browsers (Chrome, Firefox, Safari, Edge)

Use Cases

Create New Task

Preconditions:

  • Application is running
  • User is on the main page

Steps:

  1. Click "Add New Task" button
  2. Enter task title
  3. Enter task description
  4. Select task status
  5. Click "Save" button

Postconditions:

  • New task is created
  • Task appears in the task list
  • Task has unique ID
  • Success message is displayed

Test Cases

Create New Task Test

Test Case ID: TC001

Description: Verify that a new task can be created successfully

Preconditions:

  • Application is running
  • User is on the main page

Test Steps:

  1. Click "Add New Task" button
  2. Enter task title: "Complete project documentation"
  3. Enter task description: "Write comprehensive documentation for the task management application"
  4. Select status: "In Progress"
  5. Click "Save" button

Expected Results:

  • New task is created
  • Task appears in the task list
  • Task has unique ID
  • Success message is displayed

Implementation Steps

  1. Set up the development environment
  2. Create the basic application structure
  3. Implement the task creation functionality
  4. Implement the task viewing functionality
  5. Implement the task status update functionality
  6. Implement the task deletion functionality
  7. Implement the task filtering functionality
  8. Implement the task search functionality
  9. Write and execute test cases
  10. Document any issues found
