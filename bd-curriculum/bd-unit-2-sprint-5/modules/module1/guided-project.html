Guided Project: Writing and Testing Functional Requirements

Project Overview

In this guided project, you'll work on a simple task management application. You'll write functional requirements, create use cases, and develop test cases to verify the application's behavior.

Project Requirements

Task Management Application

The application should allow users to:

  • Create new tasks
  • View all tasks
  • Mark tasks as complete
  • Delete tasks

Use Cases

Create New Task

Preconditions:

  • Application is running
  • User is on the main page

Steps:

  1. Click "Add New Task" button
  2. Enter task title
  3. Enter task description
  4. Click "Save" button

Postconditions:

  • New task is created
  • Task appears in the task list
  • Task has unique ID

View All Tasks

Preconditions:

  • Application is running
  • User is on the main page

Steps:

  1. Tasks are automatically displayed on the main page

Postconditions:

  • All tasks are visible
  • Tasks are displayed in chronological order

Test Cases

Create New Task Test

Test Case ID: TC001

Description: Verify that a new task can be created successfully

Preconditions:

  • Application is running
  • User is on the main page

Test Steps:

  1. Click "Add New Task" button
  2. Enter task title: "Complete project documentation"
  3. Enter task description: "Write comprehensive documentation for the task management application"
  4. Click "Save" button

Expected Results:

  • New task is created
  • Task appears in the task list
  • Task has unique ID
  • Success message is displayed

Implementation Steps

  1. Set up the development environment
  2. Create the basic application structure
  3. Implement the task creation functionality
  4. Implement the task viewing functionality
  5. Write and execute test cases
  6. Document any issues found
