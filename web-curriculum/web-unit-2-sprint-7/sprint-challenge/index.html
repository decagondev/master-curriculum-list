Sprint Challenge - Web Unit 2 Sprint 7

Project Overview

In this sprint challenge, you will build a React application that demonstrates your understanding of React Router, Form Management, and Testing. You will create a multi-page application with form handling and comprehensive test coverage.

Sprint Topics

This sprint focused on three key areas of modern React development:

React Router

Client-side routing allows your application to update the URL and render different components without making a request to the server for a new page. Key concepts include:

  • Difference between server-side and client-side routing
  • Setting up routes and links in a React application
  • Using URL parameters to create dynamic routes
  • Programmatic navigation with useNavigate

Form Management

Forms are a fundamental part of web applications. React offers controlled components to manage form data effectively. Key concepts include:

  • Building forms with basic HTML form elements
  • Using the onChange handler to capture input values
  • Utilizing a form's submit event to work with form data
  • Handling multiple inputs in a form
  • Form validation techniques

End-to-End Testing with Cypress

End-to-end testing verifies the entire user experience from start to finish. Cypress makes it easy to write and run E2E tests. Key concepts include:

  • Importance of end-to-end testing in the development process
  • Using the Cypress GUI to write tests and interact with elements
  • Performing actions like clicking, typing, and submitting forms
  • Running tests without the UI for CI/CD pipelines

Testing Implementation

Write tests using Cypress following the "arrange, act, assert" pattern:


// Example E2E test
describe('Form submission', () => {
  it('should successfully submit the form with valid data', () => {
    // Arrange - Visit the form page
    cy.visit('/form')
    
    // Act - Fill in form fields and submit
    cy.get('input[name="username"]').type('testuser')
    cy.get('input[name="email"]').type('test@example.com')
    cy.get('form').submit()
    
    // Assert - Check we navigated to the results page
    cy.url().should('include', '/results/testuser')
    cy.contains('Successfully submitted!').should('be.visible')
  })

  it('should show validation errors for invalid input', () => {
    // Arrange
    cy.visit('/form')
    
    // Act - Submit with empty form
    cy.get('form').submit()
    
    // Assert - Check validation errors appear
    cy.contains('Username is required').should('be.visible')
    cy.contains('Email is required').should('be.visible')
  })
})
