Understanding Relational Databases

A relational database is a collection of data organized as a set of formally described tables. These tables have relationships between them, allowing data to be defined and accessed in various ways.

Core Components

  • Tables: The main building blocks that store data in rows and columns, similar to spreadsheets.
  • Records (Rows): Each individual entry in a table, representing a single instance of an entity.
  • Fields (Columns): Categories of data that define what type of information is stored.
  • Primary Keys: Unique identifiers for each record in a table, ensuring data integrity.
  • Foreign Keys: Fields that link to primary keys in other tables, establishing relationships.

Example of a simple table structure:

CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  username TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

SQL Fundamentals

SQL (Structured Query Language) is the standard language for interacting with relational databases. It allows you to create, read, update, and delete data with precision.

Key SQL Operations

  • SELECT: Retrieves data from one or more tables.
  • INSERT: Adds new records to a table.
  • UPDATE: Modifies existing records.
  • DELETE: Removes records from a table.
  • JOIN: Combines rows from multiple tables based on related columns.

Example SQL queries:

-- Select all users
SELECT * FROM users;

-- Insert a new user
INSERT INTO users (username, email) 
VALUES ('john_doe', 'john@example.com');

-- Update a user's email
UPDATE users 
SET email = 'john.doe@example.com' 
WHERE username = 'john_doe';

-- Delete a user
DELETE FROM users 
WHERE username = 'john_doe';

Working with Knex.js

Knex.js is a SQL query builder for Node.js that provides a unified interface for interacting with different database systems, including SQLite, PostgreSQL, MySQL, and others.

Benefits of Knex.js

  • Prevents SQL injection attacks through query parameterization
  • Provides a consistent API across different database systems
  • Makes complex queries more readable with a chainable interface
  • Supports migrations and seeds for database version control

Example of Knex.js setup and queries:

// Setup connection
const knex = require('knex')({
  client: 'sqlite3',
  connection: {
    filename: './dev.sqlite3'
  },
  useNullAsDefault: true
});

// Select all users
knex('users')
  .select('*')
  .then(users => {
    console.log(users);
  });

// Insert a new user
knex('users')
  .insert({
    username: 'jane_doe',
    email: 'jane@example.com'
  })
  .then(ids => {
    console.log('New user created with ID:', ids[0]);
  });

// Update a user
knex('users')
  .where({ username: 'jane_doe' })
  .update({ email: 'jane.doe@example.com' })
  .then(count => {
    console.log(`${count} user(s) updated`);
  });
