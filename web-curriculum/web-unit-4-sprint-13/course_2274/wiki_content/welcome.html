Welcome to Sprint 13!

This week we'll learn how to build web services based on the REpresentational State Transfer (REST) architectural style.

We'll learn about Node.js, a runtime environment used to execute JavaScript code outside the browser. We will also learn about the popular Express framework that makes it easy to write web applications and services on top of Node.js.

Review Learning Resources and the Unit Syllabus before you begin

Module 1

Introduction to Node.js and Express

We will introduce Node.js and Express. Discover what Node.js is and what it can and cannot do. Explore Express and look at why it's a helpful tool to use. Examine the relationship between Express and Node.js. See a high-level overview of the main features of Express. Learn how to test our Web API using a tool called Postman manually. Create our first CRUD Web API.

Module 2

Server-Side Routing with Express

Node's HTTP server uses a single request handler function for all requests. However, Express provides a way to break the application into different request handlers for each URL + HTTP method combination. This feature is called routing and is one of the main features of Express. We will use the routing feature of Express to build several endpoints. Cover different ways to pass data to a Web API. Introduce REST as a way to guide the architecture of the web. Use Express Routers to organize our code.

Module 3

Express Middleware

Express is a minimalist framework. It doesn't provide everything out of the box, but using middleware can add extra functionality to our application. In addition, using middleware offers a way to extend the features provided by the Express framework. They are implemented as small functions that handle one aspect of our application. For example, tasks like authentication and logging are commonly handled by middleware. Another benefit of middleware is that it provides an easy way to add modularity to our code.

Module 4

Web Deployment and Best Practices

The time has come to show your work to the world. In this module, we'll learn how to make out API deployable. By extracting configuration details from the application and allowing the environment to keep track of them, we will facilitate the deployment of our API to Heroku and other similar services.
