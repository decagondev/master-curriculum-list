Understanding Recursion

Recursion is a programming technique where a function calls itself to solve a problem by breaking it down into smaller instances of the same problem.

Every recursive solution requires:

  • Base Case(s): Condition(s) that terminate the recursion
  • Recursive Case(s): The function calling itself with modified input
  • Progress toward the base case: Ensuring termination
// Factorial implementation using recursion function factorial(n) { // Base case if (n === 0 || n === 1) { return 1; } // Recursive case return n * factorial(n - 1); } // Example usage console.log(factorial(5)); // 120

Recursion With Linked Lists

Recursion is particularly useful for traversing linked structures like linked lists:

// Finding the length of a linked list recursively function getLength(head) { // Base case: empty list if (head === null) { return 0; } // Recursive case: count this node + rest of list return 1 + getLength(head.next); } // Reversing a linked list recursively function reverse(head) { // Base cases: empty list or single node if (head === null || head.next === null) { return head; } // Recursive case const newHead = reverse(head.next); // Change pointers head.next.next = head; head.next = null; return newHead; }

Understanding The Call Stack

When a function calls itself, each call is added to the "call stack" until the base case is reached:

// Tracing factorial(3): factorial(3) → 3 * factorial(2) → 3 * (2 * factorial(1)) → 3 * (2 * 1) → 3 * 2 → 6

Understanding how the call stack works is crucial for debugging recursive functions and avoiding stack overflow errors.
